# RPG using godot and dotnet
This project will continue in the [GodotRpgProject2026](https://github.com/jvcmtr/GodotRpgProject2026) due to some issues.

There is not much to be seen here from a game development standpoint, but there were some cool things learned in the process:

- The [Attribute System](https://github.com/jvcmtr/GodotRpgProject2025/tree/main/_docs_rpgsys/2.%20SystemDocumentation/Attributes) was realy flexible.
- Practice a little of some lazy [OOP](https://github.com/jvcmtr/GodotRpgProject2025/tree/main/_docs_rpgsys/4.%20DeveloperNotes/SystemPlanning), including some and [niche c# stuff](https://github.com/jvcmtr/GodotRpgProject2025/tree/main/_docs_rpgsys/4.%20DeveloperNotes/Study/CSharp/Dinamic%20Type%20Polymorphism)
- Implemented my own [mapper](https://github.com/jvcmtr/GodotRpgProject2025/blob/main/DLL/Enums/EnumMapperCore.cs), witch i think it was realy easy to use
- The [Stats Models](https://github.com/jvcmtr/GodotRpgProject2025/tree/main/DLL/Stats) (stats, resource pools and modifiers) in general are realy cool and usefull.
- Learned a lot about godot and vscode, including [headless godot](https://github.com/jvcmtr/GodotRpgProject2025/tree/main/_docs_rpgsys/4.%20DeveloperNotes/Infrastructure/VSCODE_GODOT), and how add [some glue](https://github.com/jvcmtr/GodotRpgProject2025/blob/main/DLL/TestMain.cs) between the two.
- Also just created a bunch of cool little classes and [test stuff](https://github.com/jvcmtr/GodotRpgProject2025/tree/main/DLL/Test_Infra)
