# csNode Plugin for Godot

### What the Addon Does

**csNode** is a Godot Editor plugin that automatically scans a specific folder in your project (`res://DLL/nodes`) for C# scripts (`.cs`) and registers them as custom node types in the editor.

This plugin is especially useful for projects that dynamically generate or import C# scripts and want them available in the Godot editor without manual registration.

Features include:
- Automatic detection and registration of `.cs` scripts as Godot nodes.
- Prefixes custom nodes with `DLL_` for easy identification.
- Automatically refreshes when the project is rebuilt or resources change.
- Uses a default icon for all registered nodes.

> All the above features are costumizable via constants in `csNode.gd`

---

### Setup
##### 1. Install the Plugin
   Place the plugin folder in your project’s `res://addons/` directory. Make sure `csNode.gd` and `plugin.cfg` are included.

##### 2. Enable the Plugin
   Open Godot → `Project` → `Project Settings` → `Plugins` → Enable **csNode**.

##### 3. Add scripts to the Scan Folder
   By default, the plugin scans `res://DLL/nodes` for files with the `.cs` extension.
   Ensure your `.cs` scripts are stored in this folder.

##### 4. Automatic Refresh
   Whenever you rebuild your C# scripts or modify the folder, the plugin will automatically refresh the registered nodes.

---

### Customization

##### Configuration File
All the properties can be configuring by changing constants in `ress://addons/csNode/csNode.gd`.

- `SCAN_FOLDER` : The folder that will be scaned for csharp scripts
- `FILES_SUFIX` : The file extension os the script you want to register as nodes.
- `NODE_PREFIX` : This variable allows you to make so that all generated nodes have a certain prefix (for organization sake)
- `DEFAULT_ICON` : Path for the icon of the generated nodes.
